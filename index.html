◆ 프록시 서버란?

프록시 서버는 클라이언트(예: 여러분의 컴퓨터)와 다른 서버(예: 웹사이트) 사이에 위치한 중개자 역할을 합니다. 여러분이 웹사이트에 접속하려고 할 때, 요청은 먼저 프록시 서버로 보내지고, 프록시 서버가 대신 해당 웹사이트에 접속하여 응답을 받아 여러분에게 전달해줍니다.

이러한 중개 역할은 여러 가지 이점을 제공합니다:

  • 보안 강화: 실제 IP 주소를 숨겨 익명성을 제공합니다.
  • 접속 속도 향상: 자주 요청되는 콘텐츠를 캐시하여 더 빠르게 전달합니다.
  • 접근 제어: 특정 웹사이트 접속을 차단하거나 허용할 수 있습니다.

◆ 근명고등학교 LABS 프록시 구조

다음은 근명고등학교 LABS로의 안전하고 효율적인 접근을 위한 프록시 구조입니다. 이 구조는 NetBird P2P(Peer-to-Peer) 터널링 기술을 활용하여 기존의 방식보다 더욱 유연하고 안전한 연결을 보장합니다.

1. Proxy (프록시)

외부에서 접속을 시도하는 모든 인터넷 요청은 이 프록시를 통해 이루어집니다. 프록시 서버는 여러분의 요청을 받아 암호화된 터널을 통해 안전하게 전송하는 역할을 합니다.

2. NetBird P2P Tunnel (터널)

NetBird는 P2P 터널링 기술을 사용하여 프록시와 근명고등학교 LABS 사이에 안전한 통신 통로를 만듭니다. 이 터널은 모든 데이터 트래픽을 강력하게 암호화하여 중간에 가로채거나 조작하는 것을 불가능하게 합니다. NetBird 터널은 다음의 특징을 가집니다:

  • 분산형 구조: 중앙 서버를 거치지 않고 직접 통신하여 효율적입니다.
  • 양자 저항 암호화: 최신 양자 저항 암호화를 사용하여 데이터를 양자 컴퓨터의 공격에서 보호합니다.
  • 뛰어난 보안: 최신 암호화 프로토콜을 사용하여 데이터를 보호합니다.
  • 안정적인 연결: 네트워크 환경 변화에 유연하게 대처합니다.

3. 근명고등학교 LABS

접속의 최종 목적지입니다. 학교에 위치한 LABS 서버로, 프록시를 통해 안전하게 접속할 수 있습니다. 이 구조 덕분에 외부에서도 LABS 자원을 이용할 수 있습니다.
